ISSUE CORE:
- The `pvlib.irradiance.reindl` function produces `NaN` or `Inf` when Global Horizontal Irradiance (`ghi`) is zero.
- This is caused by a division by zero in the calculation of `term3`, specifically `np.sqrt(HB / ghi)`.

OBSERVED BEHAVIOR:
- Running `reindl` with `ghi=0` results in a `RuntimeWarning: invalid value encountered in scalar divide` and returns `nan`.
- Physically, if `ghi` is zero, the diffuse sky irradiance should also be zero (or at least not `NaN`).
