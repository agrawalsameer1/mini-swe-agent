ISSUE CORE:
- `pvlib.temperature.fuentes` fails when the input `poa_global` has a timezone-aware index.
- The failure occurs during the calculation of `timedelta_hours` because `np.diff()` on a timezone-aware `DatetimeIndex` returns an array of `Timedelta` objects, which cannot be cast directly to `float` via `.astype(float)`.

OBSERVED BEHAVIOR:
- `TypeError: float() argument must be a string or a real number, not 'Timedelta'`
- Occurs at `pvlib/temperature.py:602`.
- Naive indices work correctly because `np.diff` on them returns nanosecond integers (in some pandas/numpy versions) or compatible types, whereas TZ-aware indices return `Timedelta` objects.

CONFIRMED RELEVANT FILES:
- `pvlib/temperature.py`: Contains the `fuentes` function and the failing line.
