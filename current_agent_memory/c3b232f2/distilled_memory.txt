ISSUE CORE:
- `Dataset.to_json_dict(suppress_invalid_tags=True)` fails to suppress exceptions when accessing tags that contain invalid data.
- The cause is that `data_element = self[key]` is called outside the `try...except` block. In `pydicom`, accessing a key in a dataset read from a file triggers lazy decoding (`DataElement_from_raw`), which can raise `TypeError` or `ValueError` if the data is invalid for its VR (e.g., an invalid `IS` value like '2.5').

OBSERVED BEHAVIOR:
- `TypeError: Could not convert value to integer without loss` is raised during `to_json_dict` even when `suppress_invalid_tags=True`.
- The traceback shows the error occurs at `data_element = self[key]` inside `pydicom/dataset.py`.
