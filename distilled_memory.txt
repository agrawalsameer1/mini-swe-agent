ISSUE CORE:
- Rule L031 ("Avoid table aliases in from clauses and join conditions") incorrectly triggers on single-table queries that use aliases, even when no `JOIN` is present.
- The error message "Avoid using aliases in join condition" is misleading for single-table queries.
- The rule implementation in `src/sqlfluff/rules/L031.py` lacks a check for the presence of multiple tables or explicit join clauses before flagging aliases.

OBSERVED BEHAVIOR:
- Running `sqlfluff lint` on `SELECT a.[hello] FROM mytable AS a` with `--dialect tsql` results in a violation: `L031 | Avoid using aliases in join condition`.
- This behavior is also observed in other dialects (e.g., `ansi`).
- The rule attempts to "fix" the query by replacing the alias with the full table name, which may not be desired in single-table contexts.

CONFIRMED RELEVANT FILES:
- `src/sqlfluff/rules/L031.py`: Contains the logic for Rule L031.
- `test/fixtures/rules/std_rule_cases/L031.yml`: Contains existing test cases for the rule.
