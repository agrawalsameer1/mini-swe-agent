ISSUE CORE:
- `DateTime._bind_to_schema` (and by extension `Date`) raises an `AttributeError` when nested inside a container field like `List` or `Tuple`.
- The cause is that container fields pass themselves as the `schema` argument to the inner field's `_bind_to_schema` method, but `DateTime` assumes this argument has an `opts` attribute (which only `Schema` objects have).

OBSERVED BEHAVIOR:
- Instantiating a `Schema` containing `fields.List(fields.DateTime())` or `fields.List(fields.Date())` fails with:
  `AttributeError: 'List' object has no attribute 'opts'`
- Traceback points to `src/marshmallow/fields.py`, line 1117, in `_bind_to_schema`.
