ISSUE CORE:
- `pydicom` configures its logger (`pydicom`) with a `StreamHandler`, a custom `Formatter`, and sets the log level to `WARNING` automatically upon import in `pydicom/config.py`.
- This violates library best practices, as it interferes with the logging configuration of the host application.

OBSERVED BEHAVIOR:
- Importing `pydicom` adds a `StreamHandler` to the `pydicom` logger.
- The log level is forced to `WARNING` via an internal call to `debug(False)` at the end of `pydicom/config.py`.
- A reproduction script confirmed that even with a root logger configured, `pydicom` messages are handled by its own internal `StreamHandler` and level settings.

CONFIRMED RELEVANT FILES:
- `pydicom/config.py`: Contains the logger initialization, handler setup, and the `debug()` function call that forces the log level.

- None so far.
