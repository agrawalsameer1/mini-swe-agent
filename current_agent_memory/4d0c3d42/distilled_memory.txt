ISSUE CORE:
- `pvlib.tools._golden_sect_DataFrame` fails with an `Exception: Iterations exceeded maximum` when the input `lower` and `upper` bounds are equal (e.g., when GHI and DHI are both zero).
- The root cause is the calculation of `iterlimit`: when `upper - lower` is 0, `np.log(atol / 0)` results in `inf`. Since `np.log(phim1)` is negative, the resulting `iterlimit` becomes `-inf`.
- Because `iterlimit` is `-inf`, the optimization loop `while not converged and (iterations <= iterlimit)` never executes, leaving `converged = False` and triggering the exception.
