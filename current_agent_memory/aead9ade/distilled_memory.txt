ISSUE CORE:
- The SQLFluff parser fails with a `RuntimeError: Dropped elements in sequence matching!` when encountering double semicolons (e.g., `SELECT 1;;`).
- This occurs because the grammar's `match` result does not account for all input segments, causing a validation check in `check_still_complete` to fail.

OBSERVED BEHAVIOR:
- Command: `echo "select id from tbl;;" | python3 -m sqlfluff lint -`
- Error: `RuntimeError: Dropped elements in sequence matching! 'select id from tbl;;' != ';'`
- The traceback indicates the failure happens during the `parse` phase of a segment (likely `FileSegment` or `StatementSegment`).

CONFIRMED RELEVANT FILES:
- `src/sqlfluff/core/parser/helpers.py`: Contains `check_still_complete` which raises the error.
- `src/sqlfluff/core/parser/segments/base.py`: Calls `check_still_complete` after a grammar match.
- `src/sqlfluff/dialects/dialect_ansi.py`: Defines the top-level `FileSegment` and `StatementSegment` structure.
